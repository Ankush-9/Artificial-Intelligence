# Core Concepts of AI

Artificial Intelligence (AI) has been used in the industry for many years. However, recent advancements have made AI accessible to the general public through tools such as ChatGPT, Gemini, Claude, GitHub Copilot, Cursor AI, and many others.

Understanding the core concepts of AI is becoming increasingly important for modern engineers.

---

# Large Language Models (LLMs)

A **Large Language Model (LLM)** is a type of Artificial Intelligence trained on massive amounts of text data to understand, generate, and work with human language.

## Examples of LLMs

* ChatGPT
* Gemini
* Claude
* Llama

## How an LLM Works

### 1. Training

The model learns:

* Language patterns
* Grammar
* Facts
* Reasoning styles
* Relationships between words

using large datasets collected from books, websites, articles, documentation, and other text sources.

### 2. Prediction

Given some text, the model predicts the most likely next word (or token).

### 3. Generation

By repeatedly predicting the next token, the model can:

* Answer questions
* Generate code
* Summarize content
* Create documentation
* Assist with problem-solving
* Generate creative content

---

# Prompts

A **Prompt** is the input provided by the user to an LLM.

Example:

```text
Write a Dockerfile for a Maven web application.
```

The quality of the prompt directly affects the quality of the output.

> Better Prompt → Better Output

---

# Prompt Engineering

**Prompt Engineering** is the practice of writing prompts that help AI models generate accurate, relevant, and useful responses.

To obtain precise output, clearly define:

* Task
* Context
* Constraints
* Requirements
* Expected Format

## Why Prompt Engineering Matters

The more clearly you explain your requirements, the more accurate the output becomes.

### Example

Prompt:

```text
Write a Dockerfile for a Maven web application.
```

The model may provide:

* Dockerfile
* Additional explanation
* Theory

Improved Prompt:

```text
Write only the Dockerfile for a Maven web application.
Do not include explanations.
```

The model returns only the required Dockerfile.

---

# Prompt Engineering Techniques

## 1. Provide Context

Tell the model which role it should assume.

Examples:

```text
Act as a Recruiter.
Act as a DevOps Engineer.
Act as a Cloud Architect.
```

---

## 2. Use Constraints

Specify limitations and requirements.

Examples:

* Word limit
* Format
* Tone

```text
Provide output in a table format.
Explain in simple language.
Limit response to 100 words.
```

---

## 3. Use Examples

LLMs learn patterns effectively when examples are provided.

Example:

```text
Generate output similar to the following example...
```

Providing examples often improves response quality.

---

## 4. Iterative Prompting

Improve outputs through follow-up prompts.

Examples:

```text
Add more details.
Format this better.
Explain with an example.
```

Prompting is usually an iterative process.

---

## 5. Basic Prompt Formula

```text
Role + Task + Context + Constraints + Example
```

### Example

```text
Act as a DevOps Engineer.

Write a Dockerfile for a Maven web application.

Return only the Dockerfile.

Do not include explanations.
```

**Generative AI** is a branch of Artificial Intelligence that creates new content.

It can generate:

* Text
* Images
* Code
* Audio
* Video

## Examples

* ChatGPT
* Gemini
* Claude
* DALL-E
* Midjourney

Generating new content using AI models is known as **Generative AI**.

# AI Chatbots

AI Chatbots are conversational systems designed to interact with users and provide information.

## Examples

* ChatGPT
* Gemini
* Claude

## Capabilities

* Answer questions
* Explain concepts
* Generate content
* Provide recommendations

## Limitations

* Primarily focused on conversations.
* Generally do not perform actions on behalf of users.

---

# AI Agents

AI Agents are autonomous systems capable of performing actions on behalf of users with minimal human intervention.

## Characteristics

* Goal-oriented
* Autonomous
* Can interact with tools and systems
* Can perform actions without continuous user input

## Examples

### GitHub Copilot Agent

Can:

* Auto-fix Pull Request issues
* Manage repository tasks
* Suggest code changes

### AWS AIOps Agent

Can:

* Detect anomalies
* Monitor infrastructure
* Trigger scaling actions

### Kubernetes Auto-Healing Agent

Can:

* Detect failed workloads
* Restart crashed pods
* Maintain cluster health

> AI Agents are generally paid services.

# AI Assistants

AI Assistants help engineers by generating code, configurations, documentation, and suggestions.

## Examples

### GitHub Copilot

Can generate:

* Dockerfiles
* YAML files
* Terraform configurations
* Application code

### Cursor AI

Can:

* Edit codebases
* Refactor code
* Generate new features
* Assist with debugging

## Key Difference

| AI Assistant              | AI Agent                 |
| ------------------------- | ------------------------ |
| Assists users             | Performs actions         |
| Requires user supervision | Can operate autonomously |
| Suggestion-oriented       | Action-oriented          |

# Tokens and Cost Optimization

## What is a Token?

A token is the basic unit processed by an LLM.

A token may represent:

* A word
* Part of a word
* Punctuation
* Special characters

## Why Tokens Matter

Most commercial LLMs charge based on token usage.

```text
More Tokens = Higher Cost
Fewer Tokens = Lower Cost
```

Therefore:

* Precise prompts reduce unnecessary tokens.
* Better prompts improve efficiency.
* Better prompts can reduce operational costs.

> Accurate Input → Useful Output

# Enterprise LLM Usage

Many organizations use their own internal LLMs instead of public AI systems.

## Reasons

* Security
* Compliance
* Data Privacy
* Intellectual Property Protection

## Benefits

* Company data remains internal.
* Information is governed by organizational policies.
* Sensitive information is protected.

In many DevOps and enterprise environments, public AI tools may be restricted, while approved internal AI solutions are allowed.
